=== Contenu

. **Master source** : `README.md` (en) marqué en en-tête
   `<!-- master source — other languages are translations of this file -->`
. **Traductions** : les 9 autres langues portent l'en-tête
   `<!-- translated from README.md rev 0.0.1 -->`
. **Badges** : Maven Central, Gradle Plugin Portal, CI, Coverage, License
   (alignés sur le master, identiques dans toutes les langues)
. **Markers version** : `0.0.1` dans le bandeau métadonnées
   (Version · Group · Plugin ID) de chaque README
. **Switcher i18n** : ligne de liens croisés en haut de chaque README
  (ex. `🌐 Languages: **EN** | [Français](README.fr.md) | [العربية](README.ar.md) …`)
  permettant la navigation entre les 10 langues
. **RTL** : arabe (`ar`) et ourdou (`ur`) utilisent la direction RTL
  native du Markdown (rendu dépendant du viewer — GitHub/IDE)
. **Métadonnées techniques** : Java 24 · Kotlin 2.3.20 · Gradle 9.5.1
  (consommateur) / Gradle 9.5.1 (plugin interne)

=== Alignement

- Couverture linguistique identique à SLD-3 (10 langues cibles).
- Les README consommateur décrivent l'usage du plugin
  (`education.cccp.slider`, tâche `asciidoctorRevealJs`).
- Les README plugin décrivent l'intérieur du plugin
  (module layout, dépendances, tâches, tests).
- Pattern de référence : bakery-gradle BKY-I18N (README traduits first,
  câblage code ensuite).

== Impact sur la roadmap

Cette session ne cloture pas une US formelle de SLD-3 mais pose le
socle documentation i18n attendu par SLD-3.6 (Documentation +
dogfooding). La traduction des README est un prérequis implicite de
SLD-3.6. À ce titre :

- SLD-3.6 passe en *partiellement amorcé* (README i18n réalisés,
  deck multilingue de démonstration reste à faire).
- SLD-3.0 → 3.5 restent inchangés (☐ A FAIRE).
