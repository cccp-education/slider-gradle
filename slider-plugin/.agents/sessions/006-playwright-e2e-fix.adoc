== Contexte

Session 005 avait bloqué car `file://` ne peut pas résoudre les CDN externes
de reveal.js (bootstrap, jquery, reveal.js depuis cdnjs.cloudflare.com).
Le `waitForSelector(".reveal .slides")` échouait avec TimeoutError.

== Problèmes identifiés et corrigés

=== 1. file:// → HTTP local (npx serve)

Le HTML généré par asciidoctor-gradle reference des ressources CDN externes
(`<script src="//cdnjs.cloudflare.com/...">`, `<link href="...">`).
Le protocole `file://` ne peut pas les charger → le DOM reveal.js n'est pas
construit → le waitForSelector timeout.

**Solution** : démarrer `npx serve` sur le tempDir, naviguer via
`http://localhost:4333/build/docs/asciidocRevealJs/intro.html`.

Méthode `startHttpServer()` ajoutée : lance `npx serve` en Process séparé,
attend HTTP 200 dans un délai de 15s avec polling 500ms, lève une exception
si le serveur ne répond pas.

=== 2. Chemin HTML incorrect

L'asciidoctor-gradle produit les slides dans `build/docs/asciidocRevealJs/`,
pas dans `slides/misc/`. Le chemin a été corrigé :
- Avant : `slides/misc/intro.html` (n'existe pas)
- Après : `build/docs/asciidocRevealJs/intro.html`

=== 3. Strict mode violation — 4 sections

`.reveal .slides section` match 4+ `<section>` (title slide, stack parent,
Playwright E2E, Features Tested). PlaywrightAssertions en strict mode
refuse `isVisible()` sur plus d'un élément.

**Solution** : `.first()` pour cibler uniquement la première slide visible.

=== 4. Nettoyage des imports

Import `kotlin.math.absoluteValue` inutilisé retiré.
